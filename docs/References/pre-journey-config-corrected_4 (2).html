Journey Access Setup 1 selected
Select the link distribution and authentication combination that controls how users access this journey.
1
Link Distribution

Pick how each journey link is generated and who can use it.

Targeted

Generates a unique link per user. Best for onboarding specific, known individuals.

Broadcast

Generates a single shared link for all users. Best for open or bulk onboarding flows.

2
User Authentication

Decide whether users must verify their identity before starting the journey.

Authenticated

Verifies the user's identity via OTP before granting access. Users can resume an incomplete journey from where they left off.

Non-Authenticated

No identity check required. Any user can access the link, but drop-offs must restart the journey from the beginning.

Note: Without authentication, journey progress cannot be saved. If a user drops off, they must restart from screen 1. This also means partial KYC data submitted mid-journey cannot be linked to a returning user. Confirm this is appropriate for your compliance requirements before continuing.
3
OTP Delivery Channel

Choose where the one-time password is sent for authentication.

Mobile Number OTP

A one-time password is sent to the mobile number entered by the user. The user must verify the OTP to authenticate and proceed.

Email OTP Coming soon

A one-time password is sent to the email ID entered by the user. The user must verify the OTP to authenticate and proceed.
